üß† How Expression Evaluation Works

üöÇ

Shunting Yard Algorithm

Invented by Dijkstra! Converts infix (2+3) to postfix (2 3 +) using operator precedence and stacks. Named after railroad shunting yards!

üì•

Infix Notation

How humans write math: operators between operands (2 + 3). Requires precedence rules and parentheses for order.

üì§

Postfix (RPN)

Reverse Polish Notation: operators after operands (2 3 +). No parentheses needed! Easy for computers to evaluate with one stack.

‚öôÔ∏è

Stack-Based Evaluation

Postfix is evaluated left-to-right with a stack: push numbers, pop for operators, push result. Simple and efficient!

üí° Fun Facts

üöÇ Why "Shunting Yard"?

Dijkstra named it after railroad shunting yards where trains are sorted! Operators move between stacks like train cars being rearranged.

üßÆ HP Calculators

HP's famous calculators use RPN (postfix) directly! Users love it because it's faster - no = key needed!

‚ö° One Pass

Shunting Yard converts infix to postfix in a single left-to-right pass. O(n) time, incredibly efficient!
