üß† How Maze Algorithms Work

üèóÔ∏è

Maze Generation (DFS)

Uses recursive backtracking to carve paths. Start at a cell, randomly visit neighbors, backtrack when stuck. Creates perfect mazes with one solution!

üîµ

DFS Solving

Depth-First Search: Goes as deep as possible along each path before backtracking. Fast but doesn't guarantee shortest path.

üü¢

BFS Solving

Breadth-First Search: Explores all neighbors at current distance before going deeper. Always finds shortest path in unweighted mazes!

üü°

A* Solving

Heuristic search using f(n) = g(n) + h(n). Combines actual distance traveled with estimated distance to goal. Most efficient pathfinding!

üìù How Each Algorithm Works

üèóÔ∏è Maze Generation (Recursive Backtracking)

1
Start at random cell

Mark it as visited and add to stack

2
Choose random unvisited neighbor

Remove wall between current and neighbor

3
Move to neighbor, repeat

Continue until no unvisited neighbors

4
Backtrack when stuck

Pop from stack, try different path

üîµ DFS (Depth-First Search)

1
Use Stack (LIFO)

Push start cell onto stack

2
Pop and explore

Pop cell, mark as visited, check if goal

3
Push all unvisited neighbors

Goes deep along one path before trying others

4
Repeat until goal found

Naturally backtracks via stack

üü¢ BFS (Breadth-First Search)

1
Use Queue (FIFO)

Enqueue start cell

2
Dequeue and explore

Dequeue cell, mark visited, check if goal

3
Enqueue all unvisited neighbors

Explores all cells at distance d before d+1

4
Guaranteed shortest path

First time reaching goal is shortest route

üü° A* (A-Star Search)

1
Use Priority Queue

Order by f(n) = g(n) + h(n)

2
g(n) = actual cost from start

Distance traveled so far

3
h(n) = heuristic to goal

Manhattan distance (|x1-x2| + |y1-y2|)

4
Always choose lowest f(n)

Intelligently focuses on promising paths
